HTML Fundamentals & File Paths

Step 1: The Fundamental Truth

A website is not a single file. It is a collection of files (HTML, CSS, JS, images) organized in folders.

Step 2: The Core Problem

If index.html needs to display logo.png, it needs an exact, unambiguous address. Browsers use file paths to locate files.

Relative File Paths (Most Important)

Paths given relative to the current file location. You will use this 99% of the time.

  1. Same folder: Just use the filename (e.g. about.html).
  2. Subfolder: Include folder name with slash (e.g. images/logo.png).
  3. Parent folder: Use ../ to go up one level (e.g. ../images/logo.png).

HTML Boilerplate (Essential Template)

The boilerplate provides essential setup:

  • <!DOCTYPE html> tells browser to use HTML5 standards mode.
  • <html lang="en"> declares the page language for SEO and accessibility.
  • <meta charset="UTF-8"> sets character encoding to avoid garbled text.
  • <meta name="viewport" ...> ensures responsive layout on mobile devices.
  • <title> sets browser tab title and SEO info.

Grouping Content with <div>

The <div> is a generic container to group HTML elements so they can be treated as one unit.

<div>
  <img src="avatar.png" alt="Avatar">
  <h2>Arjun Kumar</h2>
  <p>Loves HTML.</p>
</div>

Labels: id and class

Use id for unique elements and class for reusable groups.

<div id="main-header">...</div>
<div class="profile-card">...</div>

Inline Grouping: <span>

Use <span> for inline styling within text without breaking the flow.

<p>This is very <span class="highlight">important</span> info.</p>
